
# 롤 상식 퀴즈 (Streamlit Project)

## 1. 프로젝트 개요
이 프로젝트는 **Python + Streamlit**으로 구현한 작은 웹 애플리케이션입니다.  
주제는 **롤 상식 퀴즈**이며, 다음 3가지 필수 기능을 모두 포함합니다.

- 로그인 기능
- 캐싱 기능
- 퀴즈 기능

또한 단순 정답 개수만 보여주는 것이 아니라,
**챔피언 상식 / 스킬 지식 / 포지션 감각 / 세계관 이해**의 4개 하위 카테고리별 결과를 리포트로 제공합니다.

---

## 2. 과제 필수 조건 반영
### 첫 화면
앱 첫 실행 화면에 아래 정보가 바로 표시됩니다.
- 학번: 2021204034
- 이름: 백다인

### 로그인 기능
- 사용자명만 입력하는 간단 로그인 방식
- 미리 정의된 사용자명 목록과 비교
- 로그인 성공 / 실패 처리
- 세션 상태 유지

### 캐싱 기능
- `st.cache_data`를 사용해 `questions.json` 로딩 + 데이터 검증 + 전처리 결과를 캐싱
- 처음 실행 시에는 로딩이 보이지만, 이후에는 캐시 재사용으로 빠르게 동작
- 화면에 **캐시 생성 시각**을 보여주어 캐싱 적용을 직접 설명 가능

### 퀴즈 기능
- 객관식 + O/X 혼합
- 문항 수 선택: 10 / 20 / 50 / 100
- 타이머 선택: 없음 / 15초 / 30초
- 최종 점수, 정답률, 카테고리별 리포트, 틀린 문제 복습 제공

## 4. 실행 방법

### 1) 가상환경 생성
```bash
python -m venv venv
```

### 2) 가상환경 활성화
#### Windows
```bash
venv\Scripts\activate
```

#### Linux / macOS
```bash
source venv/bin/activate
```

### 3) 라이브러리 설치
```bash
pip install -r requirements.txt
```

### 4) 실행
```bash
streamlit run app.py
```

## 6. 차별화 포인트
이 프로젝트는 단순한 O/X 퀴즈 앱이 아니라, 다음 요소를 추가해 차별화했습니다.

- **롤 기본 상식 + 롤 세계관(lore)**를 함께 다룸
- **객관식 + O/X 혼합**
- **문항 수 선택 기능**
- **문제별 타이머 기능**
- **카테고리별 결과 리포트**
- **틀린 문제만 다시 확인 가능한 복습 화면**
- **다크 테마 + 게임 UI 느낌의 디자인**

## 7. 캐싱을 왜 적용했는가?
퀴즈 앱은 실행할 때마다 문제 데이터를 불러오고, 그 데이터가 정상 형식인지 검증해야 합니다.  
이 과정은 매번 반복할 필요가 없으므로 `st.cache_data`를 사용했습니다.

즉, 캐싱의 목적은 다음과 같습니다.
- JSON 데이터 로딩 시간 절약
- 반복 전처리 방지
- 앱 응답 속도 개선

과제 시연 때는 다음처럼 설명하면 됩니다.

> "문제 파일을 불러오고 구조를 검사하는 과정은 매 실행마다 반복될 필요가 없어서 Streamlit 캐싱을 적용했습니다.  
> 처음에는 약간 로딩되지만, 같은 데이터는 캐시에서 재사용되어 더 빠르게 동작합니다."
