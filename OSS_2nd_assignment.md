## 자기소개

저는 데이터의 안정적인 저장과 효율적인 활용을 책임지는 **DBA(Database Administrator)** 를 목표로 하는 예비 데이터 엔지니어입니다.  
데이터베이스의 성능 최적화, 백업 및 복구, 보안 관리, 장애 대응, 운영 자동화에 관심이 많으며, 단순히 데이터를 저장하는 것을 넘어 **서비스의 신뢰성과 지속 가능성을 높이는 DBA**로 성장하고자 합니다.  
미래에는 대규모 서비스 환경에서도 안정적으로 데이터베이스를 운영하고, 문제를 빠르게 진단하고 해결할 수 있는 실무형 DBA가 되는 것이 목표입니다.

## 프로젝트 경험

### 1. 데이터베이스 모니터링 대시보드 개발
**역할:** 백엔드 및 모니터링 로직 구현
- CPU 사용률, 메모리 사용량, 활성 세션 수, 슬로우 쿼리 현황을 확인할 수 있는 대시보드를 개발함
- 데이터베이스 운영 상태를 시각화하여 이상 징후를 빠르게 파악할 수 있도록 구성함
- 트랜잭션 지연이 일정 수준을 넘을 경우 경고를 표시하는 로직을 설계함
- **기술 스택:** Python, SQL, Streamlit, MySQL

### 2. 학사관리 시스템 SQL 튜닝 프로젝트
**역할:** 데이터베이스 성능 분석 및 최적화
- 학사관리 서비스의 비효율적인 조인 및 서브쿼리를 분석함
- 인덱스 설계와 SQL 재작성으로 조회 속도를 개선함
- 튜닝 전후 실행 계획을 비교하여 성능 향상 결과를 정리함
- 향후 유지보수를 위한 SQL 튜닝 보고서를 작성함
- **기술 스택:** MySQL, SQL, EXPLAIN, Index Design

### 3. 백업 및 복구 시뮬레이션 프로젝트
**역할:** 데이터베이스 안정성 관리 설계
- 장애 상황을 가정한 데이터베이스 백업 및 복구 시나리오를 설계함
- 전체 백업, 증분 백업, 복구 절차를 직접 테스트함
- 장애 발생 시 다운타임을 최소화할 수 있도록 복구 절차를 표준화함
- 실습용 장애 대응 체크리스트를 제작함
- **기술 스택:** PostgreSQL, Bash, Linux

### 4. 대학 도서관 데이터베이스 설계 프로젝트
**역할:** 데이터베이스 설계 및 구현
- 대학 도서관 서비스를 위한 ERD 및 관계형 스키마를 설계함
- 정규화를 통해 데이터 중복을 줄이고 무결성을 높임
- 도서 검색, 예약, 연체 관리 기능을 위한 SQL을 작성함
- 사용자 및 대출 기록 관리가 가능하도록 제약 조건을 설계함
- **기술 스택:** ERD, SQL, PostgreSQL
